### Security
- **Backend — vá 7 vấn đề bảo mật phát hiện qua review website (2026-07-15)**: rà lại backend + frontend, tìm và sửa các lỗ hổng thật (không đụng schema, không đụng GroupFlow extension):
  1. **Leo thang quyền qua token GroupFlow (nghiêm trọng)**: các route `/api/user-auth/admin/*` (`userAuth.js`) chỉ dùng `authenticate`, không check role — mà `authenticate` (`middleware/auth.js`) trước đây chấp nhận cả token của `group_user` (ký với `type:'user_account'`, dùng chung `JWT_SECRET` + bảng `users`). Hệ quả: bất kỳ ai tự đăng ký free qua `/api/user-auth/register` (route public) đều xem được toàn bộ user + email + SĐT + license key, tự nâng plan, khoá/mở key người khác, xoá user bất kỳ. Fix 2 lớp: (a) `authenticate` từ chối dứt khoát token có `type:'user_account'` (token này CHỈ dành cho `/api/user-auth/*` qua `requireUserAuth`); (b) thêm `canManageUsers` vào cả 5 route admin trong `userAuth.js`.
  2. **Path traversal đọc file tuỳ ý (nghiêm trọng)**: `resolveLocalImagePath()` (`mediaStorage.js`) `path.join(publicRoot, ref)` với `ref` từ query người dùng (`GET /api/media/image`) mà không chặn `../` — gọi `?ref=../backend/.env` tải được `.env` (mật khẩu DB + JWT_SECRET). Fix: `path.resolve` rồi bắt buộc kết quả nằm gọn trong `publicRoot`.
  3. **JWT_SECRET fallback mặc định (nghiêm trọng)**: `'replace_with_strong_secret'` rải ở 3 file — deploy quên đặt secret thì ai cũng tự ký được token super_admin. Fix: `NODE_ENV=production` mà thiếu/để mặc định `JWT_SECRET` thì `process.exit(1)` ngay lúc khởi động (`app.js`).
  4. **Không có rate limit đăng nhập/đăng ký**: `/api/auth/login`, `/api/user-auth/login`, `/api/user-auth/register` không qua limiter nào — brute-force mật khẩu admin + spam đăng ký thoải mái. Thêm `authLimiter` (20 req/15 phút/IP, `middleware/rateLimit.js`) cho cả 3.
  5. **Upload ảnh giả mạo → stored XSS**: `upload.js` không giới hạn dung lượng, đuôi file lấy thẳng từ `originalname`, validate chỉ check mimetype client tự khai và fail cũng không xoá file đã ghi. Upload `evil.html` khai `image/png` → serve tại `/images/evil.html` → XSS trên domain (token ở localStorage). Fix: `fileFilter` whitelist mimetype, đuôi file suy từ mimetype (không tin `originalname`), `limits.fileSize`, xoá file khi validate fail, tên file thêm chuỗi random chống trùng.
  6. **Activity log ghi thô secret**: `sanitizeBody` (`activityLog.js`) chỉ redact 5 field, không đệ quy — `response` lồng bên trong chứa `key_value`/`client_secret`/`refresh_token` bị ghi thô vào `activity_logs`. Fix: mở rộng danh sách field nhạy cảm + redact đệ quy.
  7. **`PATCH /admin/users/:id` không validate input**: ghi thẳng `plan`/`key_status`/`expires_at` vào DB. Fix: whitelist `plan` (free/pro/enterprise), `key_status` (active/suspended), validate định dạng `expires_at`.
